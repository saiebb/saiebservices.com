🔧 XAMPP MySQL Fix

حل مشكلة توقف MySQL في XAMPP

دليل شامل لحل مشكلة "MySQL shutdown unexpectedly"

⚠️ تحذير: قم بعمل نسخة احتياطية من قاعدة البيانات قبل تطبيق أي حل!
1
إعادة تشغيل الخدمات
  • أغلق XAMPP Control Panel تماماً
  • افتح Task Manager (Ctrl + Shift + Esc)
  • ابحث عن عمليات mysql.exe و mysqld.exe وأنهها
  • أعد تشغيل XAMPP Control Panel كـ Administrator
  • ابدأ Apache ثم MySQL
2
تغيير منفذ MySQL
  • في XAMPP Control Panel، اضغط على "Config" بجانب MySQL
  • اختر "my.ini"
  • ابحث عن السطر: port=3306
  • غيره إلى: port=3307
  • احفظ الملف وأعد تشغيل MySQL
# في ملف my.ini [mysqld] port=3307 [mysql] port=3307
3
حذف ملفات Log المعطوبة
  • اذهب إلى مجلد: C:\xampp\mysql\data\
  • ابحث عن الملفات: ib_logfile0 و ib_logfile1
  • احذف هذين الملفين (بعد إيقاف MySQL)
  • أعد تشغيل MySQL
تحذير: تأكد من إيقاف MySQL تماماً قبل حذف هذه الملفات!
4
إصلاح قاعدة البيانات
  • افتح Command Prompt كـ Administrator
  • اذهب إلى مجلد XAMPP: cd C:\xampp\mysql\bin
  • نفذ الأمر التالي:
mysqld --console --skip-grant-tables --skip-external-locking
  • في نافذة أخرى، نفذ:
mysql -u root FLUSH PRIVILEGES; EXIT;
5
استعادة قاعدة البيانات
  • انسخ مجلد قاعدة البيانات من: C:\xampp\mysql\data\
  • أعد تثبيت XAMPP
  • استعد مجلد قاعدة البيانات
  • استورد ملف SQL إذا لزم الأمر
